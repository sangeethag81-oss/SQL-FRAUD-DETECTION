# Financial Fraud Detection using SQL

## Overview
This project analyzes banking transaction data to identify fraudulent activities using SQL.

## Tools Used
- MySQL Workbench 8.0
- SQL

## Dataset
The dataset contains transaction details such as amount, balances, and fraud indicators.

## Analysis Performed
- Fraud vs non-fraud transaction analysis
- Fraud by transaction type
- Missed fraud detection
- Balance inconsistency checks

## Key Insights
- Fraud mainly occurs in TRANSFER and CASH_OUT transactions
- Many fraudulent transactions are not flagged by systems
- Balance anomalies are strong fraud indicators
